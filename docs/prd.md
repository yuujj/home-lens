**버전**: 1.0
**작성일**: 2026-03-13
**작성자**: 사내 해커톤 참가자 (IT개발본부)

## 1. 배경 및 문제 정의

### 1-1. 문제 상황

전세 사기 피해는 2022년부터 급증하여 사회적 문제로 대두되었다. 비전문 임차인이 수천만~수억 원의 전세금을 걸고 계약을 체결하지만, 등기부등본 해석·시세 파악·정책 대출 조회를 모두 직접 해야 하는 구조적 정보 비대칭이 존재한다.

| # | 핵심 문제 | 현재 대안 | 대안의 한계 |
|---|----------|----------|-----------|
| 1 | **등기부등본 해석 불가** — 비전문가가 가압류·근저당·신탁 등 법률 용어를 이해하지 못해 위험 계약 체결 | 공인중개사 의존 | 이해충돌 가능성, 비용 발생 |
| 2 | **전세가율 판단 불가** — 시세 대비 전세금 비율을 직접 조사하려면 여러 사이트를 돌아다녀야 함 | 네이버부동산/KB시세 수동 검색 | 시간 소요(수시간), 정확도 낮음 |
| 3 | **정책 대출 정보 비대칭** — 7개 이상 정책 대출 상품의 자격·한도·금리를 비교하기 어려움 | 은행 방문, 블로그 검색 | 파편적 정보, 최신성 부족 |

### 1-2. 기존 서비스와의 차별점

| 서비스 | 시세 조회 | AI 등기 분석 | 사기 위험도 | 정책 대출 매칭 |
|--------|:--------:|:-----------:|:----------:|:------------:|

| **이 집 괜찮아?** | **O** | **O (AI)** | **O** | **O** |

## 2. 목표 (Goals)

### 2-1. 제품 목표

> **"등기부등본 붙여넣기 한 번으로, 전세사기 위험도·시세분석·정책대출까지 2분 안에"**

- 비전문가가 전세 계약 안전성을 2분 이내에 스스로 확인할 수 있도록 한다.
- AI 기반 등기부등본 해석으로 법률·부동산 용어를 평이한 언어로 변환한다.
- 시세 조회 + 사기 위험도 + 대출 매칭을 단일 서비스에서 제공한다.

### 2-2. 해커톤 성공 기준

| 지표 | 목표 |
|------|------|
| 분석 완료 시간 | < 2분 (주소 입력 ~ 종합 결과 확인) |
| 실제 주소 기반 데모 동작 | 필수 |
| AI 등기부 분석 정확도 | 주요 위험 키워드(가압류·근저당·신탁) 탐지 > 80% |
| API 응답 시간 | 시세 분석 P95 < 5초 |
| 배포 URL 접속 가능 | Vercel + Railway 배포 필수 |

### 2-3. 비목표 (Non-Goals)

- 등기부등본 자동 열람 (대법원 인터넷등기소 API 없음 — 사용자 직접 붙여넣기로 대체)
- 법률 자문 또는 중개 행위 (면책 고지 필수)
- 실시간 등기 변동 모니터링 (MVP 제외, 향후 기능)
- 청약 리스트업 (MVP 제외)

## 3. 대상 사용자 (Customer Segments)

- **사회초년생/청년 임차인** (20대 후반~30대 초반)

| 디지털 접근성 | 높음 (웹/모바일 친숙) |

## 4. 핵심 기능 명세 (Features)

### F001 — 시세 기반 전세가율 진단

**우선순위**: P0
**요약**: 주소 입력 시 공공 실거래가 API로 전세가율을 자동 계산하고 5단계 등급으로 표시한다.

#### 사용자 흐름

1. 주소 입력 (도로명주소 자동완성)
2. 주택 유형 선택 (아파트/빌라/오피스텔/단독)
3. 전용면적·전세보증금 입력
4. [분석 시작] 클릭 → 5초 이내 결과 표시

#### 비즈니스 로직

```
1. 도로명주소 API → 법정동 코드 5자리(lawd_cd) 추출
2. RTMS 매매 API + RTMS 전세 API 호출 (최근 6개월, 유사 면적 ±10%)
3. 해제 건(해제여부='Y') 제거 후 중간값(median) 산출
   - 거래 10건↑: median
   - 거래 1~9건: 가중평균 + "데이터 적음" 경고
   - 거래 없음: 공시가격 ÷ 현실화율(0.69) + "추정치" 표시
4. 전세가율 = 전세가 / 매매가 × 100
5. 5단계 등급 판정
```

#### 전세가율 등급 기준

| 전세가율 | 등급 | 색상 |
|---------|------|------|
| ~60% | 안전 | 초록 |
| 60~70% | 양호 | 연두 |
| 70~80% | 주의 | 노랑 |
| 80~90% | 위험 | 주황 |
| 90%↑ | 매우 위험 | 빨강 |

#### 입력 스펙

| 필드 | 타입 | 필수 | 검증 |
|------|------|------|------|
| `address` | string | Y | 최소 5자, 도로명주소 API 확인 |
| `housing_type` | enum | Y | apt / rh / sh / offi |
| `exclusive_area_m2` | float | Y | > 0 |
| `listed_jeonse_price` | int | Y | > 0 (만원 단위) |

#### 출력 스펙

```json
{
  "market_trade_price": 85000,
  "market_jeonse_price": 52000,
  "jeonse_ratio": 61.2,
  "jeonse_grade": "양호",
  "market_data_confidence": "high",
  "price_trend": "보합",
  "price_trend_pct": -1.2
}
```

#### 예외 처리

- RTMS API 응답 없음 → 공시가격 fallback, "추정치" 레이블 표시
- 빌라(rh) → "시세 불투명 주의" 경고 추가 표시
- 다가구(sh) → "선순위 임차인 합산 필요" 안내 표시

### F002 — AI 등기부등본 분석 및 사기 위험도 스코어링

**우선순위**: P0
**요약**: 등기부등본 텍스트를 붙여넣으면 Claude AI가 자동 해석하여 위험도 점수(0~100)와 위험 플래그를 출력한다.

#### 사용자 흐름

1. 인터넷등기소(iros.go.kr)에서 등기부등본 전체 텍스트 복사
2. 서비스 입력창에 붙여넣기
3. [분석] 클릭 → AI 해석 결과 즉시 표시

#### 비즈니스 로직

**Step 1 — Claude API 등기부 파싱**

```
입력: 등기부등본 원문 텍스트
출력: {
  senior_mortgage_amount: int,    # 선순위 근저당 채권최고액 합계 (만원)
  has_attachment: bool,           # 가압류
  has_provisional_attachment: bool, # 가처분
  has_auction: bool,              # 경매개시결정
  has_trust: bool,                # 신탁등기
  has_lease_registration: bool,   # 임차권등기
  owner_acquired_recently: bool,  # 최근 6개월 내 취득
  is_corporate_owner: bool        # 법인 소유
}
```

**Step 2 — 스코어링 (0~100점, 높을수록 위험)**

| 항목 | 세부 기준 | 배점 |
|------|---------|------|
| 전세가율 점수 | ~60%: 0점 / 60~70%: 10점 / 70~80%: 20점 / 80%↑: 30점 | 0~30 |
| 근저당비율 점수 | 실효비율=(근저당+전세금)/매매가 / ~70%: 0점 / 70~80%: 15점 / 80%↑: 30점 | 0~30 |
| 등기 위험키워드 | 가압류: +10 / 가처분·경매: +20 / 신탁: +10 / 임차권등기: +15 / 가등기: +8 | 0~20 |
| 임대인 신뢰도 | 최근 6개월 취득: +5 / 법인 소유: +5 / 체납 이력: +10 | 0~20 |

**Step 3 — 등급 판정**

| 점수 | 등급 | 색상 |
|------|------|------|
| 0~30 | 안전 | 초록 |
| 31~50 | 주의 | 노랑 |
| 51~70 | 위험 | 주황 |
| 71~100 | 매우 위험 | 빨강 |

#### 출력 스펙

```json
{
  "fraud_score": 42,
  "fraud_grade": "주의",
  "fraud_flags": [
    "선순위 근저당 1억 2천만원 존재",
    "전세가율 72.3% — 주의 구간",
    "실효 전세가율 83.4% — 위험 구간"
  ],
  "registry_parsed": {
    "senior_mortgage_amount": 12000,
    "has_attachment": false,
    "has_trust": false,
    "has_lease_registration": false
  },
  "checklist_items": [
    { "item": "납세증명서(국세) 요청", "completed": false, "priority": "필수" },
    { "item": "전세보증금반환보증 가입 가능 여부 확인", "completed": false, "priority": "필수" }
  ]
}
```

#### Claude API 프롬프트 설계 원칙

- 등기부등본 원문을 청크 단위로 분할하여 토큰 비용 최소화
- 갑구(소유권)/을구(저당권) 섹션 분리 파싱
- 구조화된 JSON 출력 강제 (tool_use 또는 json_mode 활용)
- 면책 문구 포함 고지 ("AI 해석은 참고용이며...")

### F003 — 정책 대출 자격 판단 및 한도 계산

**우선순위**: P1
**요약**: 연소득·자산·나이 등 입력 시 이용 가능한 정책 대출 상품과 예상 한도·금리를 즉시 제시한다.

#### 사용자 흐름

1. 대출 목적 선택 (구입/전세)
2. 연소득, 나이, 혼인 여부, 자녀 수, 순자산 입력
3. 주택가격 또는 전세보증금 입력
4. 결과: 이용 가능 상품 목록 (금리 낮은 순 정렬)

#### 지원 상품 (7개)

| 상품명 | 용도 | 최대 한도 | LTV |
|--------|------|---------|-----|
| 청년전용 버팀목전세대출 | 전세 | 2억원 | 80% |
| 버팀목전세대출 (일반) | 전세 | 1.2억원 | 80% |
| 버팀목전세대출 (신혼) | 전세 | 2억원 | 80% |
| 신생아특례 버팀목 | 전세 | 2.4억원 | 80% |
| 디딤돌대출 | 구입 | 4억원 | 70~80% |
| 신생아특례 디딤돌 | 구입 | 5억원 | 70~80% |
| 보금자리론 | 구입 | 4.2억원 | 70% |

#### 한도 계산 공식

```
전세 상품: MIN(전세보증금 × LTV, 상품 최대 한도)
구입 상품: MIN(주택가격 × LTV, DTI 기반 한도, 상품 최대 한도)

DTI 기반 한도 = (연소득 × DTI한도 - 기타대출이자) / 연간원리금계수
```

#### 출력 스펙

```json
{
  "eligible_loans": [
    {
      "product_name": "청년전용 버팀목전세대출",
      "max_limit": 14400,
      "rate_min": 1.3,
      "rate_max": 2.1,
      "rate_with_benefit": 1.8,
      "ltv": 0.80,
      "monthly_payment_estimate": 60,
      "notes": ["만 19~34세 요건 충족", "지방 소재 시 0.2%p 추가 인하"]
    }
  ],
  "ineligible_loans": [
    { "product_name": "신생아특례 버팀목", "reason": "2년 이내 출산/입양 요건 미충족" }
  ]
}
```

### F004 — 종합 분석 결과 대시보드

**우선순위**: P0
**요약**: F001~F003 분석 결과를 3개 탭으로 구성된 결과 화면에 통합 표시한다.

#### 화면 구조

```
┌─────────────────────────────────────┐
│  종합 위험도: 42점 (주의)  ████░░ 42%  │
├──────────┬──────────┬───────────────┤
│ 시세 분석 │ 사기위험도 │   정책 대출    │
├──────────┴──────────┴───────────────┤
│  [탭 선택에 따라 상세 내용 표시]       │
└─────────────────────────────────────┘
│  ⚠️ 본 결과는 참고용이며 법적·금융     │
│  자문이 아닙니다.                     │
└─────────────────────────────────────┘
```

#### 탭 1 — 시세 분석

- 전세가율 게이지 + 등급 배지
- 추정 매매가 / 추정 전세가 표시
- 데이터 신뢰도 배지 (높음/중간/낮음/추정치)
- 3개월 가격 트렌드 (상승/보합/하락)

#### 탭 2 — 사기 위험도

- 위험도 점수 (0~100) + 4단계 등급
- 위험 플래그 목록 (발견된 위험 요인)
- 미완료 체크리스트 항목 하이라이트
- 등기부 해석 요약 (근저당·가압류 등)

#### 탭 3 — 정책 대출 (선택 단계)

- 이용 가능 상품 카드 리스트 (금리 낮은 순)
- 예상 한도 / 금리 범위 / 월 상환액 추정
- 자격 미충족 상품의 불가 사유 표시

## 5. 사용자 플로우 (2분 완료 목표)

```
[Step 1 — 30초] 주소 입력
  → 도로명주소 자동완성
  → 주택 유형 선택
  → 전세보증금 입력

[자동 처리 — 5초] 시세 분석
  → RTMS API 조회 → 전세가율 계산

[Step 2 — 60초] 등기부등본 붙여넣기
  → 인터넷등기소에서 텍스트 복사
  → 붙여넣기 → Claude 자동 해석

[결과 표시 — 즉시]
  → 종합 위험도 점수
  → 탭별 상세 내용

[Step 3 — 30초, 선택] 정책 대출 조회
  → 소득·나이 입력 → 이용 가능 상품 목록
```

## 9. 환경 변수

```
ANTHROPIC_API_KEY=       # Claude API (등기부 해석)
```

```
RONE_API_KEY=            # R-ONE 부동산원 (시세 트렌드)
BACKEND_URL=             # 프론트 → 백엔드 호출 URL
CORS_ORIGINS=            # 허용 출처
```

## 11. 데이터 제약 및 대응

| 항목 | 제약 | 대응 방법 |
|------|------|---------|
| 등기부등본 | 공개 API 없음, 건당 700원 유료 열람 | 사용자 텍스트 붙여넣기 → Claude 해석 |

| 빌라 시세 | 거래 희소 → 정확도 낮음 | "시세 불투명 — 고위험 신호" 경고 표시 |
| 납세증명서 | 직접 발급 필요 | 체크리스트 항목 + 홈택스 링크 안내 |
| 규제지역 현황 | API 없음, 수시 변경 | 정적 DB + 최신 정보 확인 링크 제공 |
| 정책 대출 금리 | 국토부 고시 수시 변경 | "최신 금리는 공식 사이트 확인" 안내 |

## 12. 법적 고려사항

- **면책 고지 (필수)**: 모든 결과 화면 하단 — "본 결과는 참고용이며 법적·금융 자문이 아닙니다. 실제 계약 전 전문가 상담 및 공문서 직접 확인을 권장합니다."
- **공인중개사법**: 중개 행위로 오인되지 않도록 "정보 제공 서비스"임을 명시
- **개인정보**: 소득·자산 등 민감 정보는 서버 미저장 원칙 (로컬 처리 권장)
- **공공데이터**: 이용조건 준수, 출처 명시 필수
- **Claude API**: 등기부등본 개인정보 전송 시 Anthropic 이용약관 확인
